# トップページと旅の共有

## トップページ

ドメインルート `/` はサーバーの旅一覧を取得しない。現在のブラウザで開いた旅だけを WebStorage (`localStorage`) から読み、最終アクセス順に表示する。サーバー上の全旅行を管理する画面は `/trips` に分離する。

履歴には旅ID、表示名、日付、再訪用パス、最終アクセス日時を最大30件保存する。共有リンクから開いた旅は共有パス `/s/:token` を再訪先として残す。

## 共有リンク

各旅の「リンクを共有」から推測困難なトークンを持つURLを作成し、クリップボードへコピーできる。合言葉は任意で、設定時は scrypt の salt/hash だけを保存する。

合言葉つきリンクを開いた場合、照合に成功するまで旅ID・タイトル・日付を返さない。成功後は通常の旅ページへ移動し、この端末のWebStorage履歴に共有リンクを記録する。
